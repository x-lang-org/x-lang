# 附录 A - 关键字

以下列表包含 X 语言中保留的关键字，因此它们不能用作标识符（如变量名、函数名等），除非在原始标识符形式中。

本附录基于 X 语言的宪法性文件（DESIGN_GOALS.md 和 README.md）。

## 当前使用的关键字

以下关键字具有当前描述的功能。

### 变量和绑定关键字

| 关键字 | 说明 |
|--------|------|
| `let` | 声明不可变变量绑定 |
| `mutable` | 声明可变绑定或可变引用（与 `let` 配合使用） |

| `virtual` | 虚方法声明 |

## 关键字和类型设计原则

X 语言的关键字和类型设计遵循以下原则（来自 DESIGN_GOALS.md）：

1. **全称，不缩写**：`function` 而非 `fn`/`func`，`integer` 而非 `int`，`boolean` 而非 `bool`
2. **基础类型小写**：`integer`、`float`、`boolean`、`string`、`character` 等基础类型使用小写
3. **含义自明**：读到关键字就知道它做什么，无需查文档
4. **来自自然英语**：优先使用普通英语词汇，而非编程术语黑话
5. **main 函数可选**：可以像 Swift 一样直接在文件顶层编写代码，不需要 `main` 函数

**设计准则**：代码应该像散文一样可读。关键字是语言的词汇表——每个词都必须是真正的英语单词，且含义精确。

## 原始标识符

原始标识符是一种语法，它允许你使用通常是关键字的词作为标识符。这对于与旧版代码的互操作性很有用，该代码是在某些词成为关键字之前编写的，或者与使用不是 X 语言关键字但另一种语言中的关键字的库进行交互。

原始标识符的语法使用反引号 `` ` `` 来转义关键字：

```x
// `let` 是原始标识符，表示变量名 "let"
let `let` = 5
```

## 保留用于将来使用的关键字

以下关键字在 X 语言中目前没有功能，但为了将来可能的使用而保留。

- `enum` - 声明枚举（目前使用 `type Name = Variant1 | Variant2`）
- `struct` - 声明结构体（目前使用记录语法）
- `union` - 声明 union
- `dyn` - 动态分发
- `abstract` - 声明抽象类或方法
- `static` - 全局生命周期或静态项
- `ref` - 通过引用绑定
- `move` - 闭包获取其环境的所有权
- `match` - 模式匹配（目前使用 `when`/`is`）
- `default` - 默认值或匹配分支
- `yield` - 从迭代器生成值
- `do` - do-while 循环或 monadic 绑定
- `package` - 包声明
- `external` - 外部函数或块
- `unsafe` - 表示不安全的代码
- `instanceof` - 运行时类型检查
- `typeof` - 获取类型
- `sizeof` - 获取类型大小
- `alignof` - 获取类型对齐
