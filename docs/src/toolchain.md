# X语言工具链

X语言提供了一套完整的工具链，帮助开发者高效地构建、测试和部署X语言项目。本文档详细介绍X语言的工具链组件，包括命令行工具、构建系统、测试框架和调试工具。

## 命令行工具

X语言的命令行工具是 `x`，它是一个功能强大的命令行界面，提供了丰富的命令来管理X语言项目。

### 基本用法

```bash
x [命令] [选项]
```

### 全局选项

- `--verbose` 或 `-v`：使用详细输出
- `--quiet` 或 `-q`：不输出日志信息
- `--color`：控制颜色输出（auto, always, never）
- `--directory` 或 `-C`：在指定目录中运行

### 命令分类

#### 构建命令

| 命令 | 描述 | 主要选项 |
|------|------|----------|
| `build` | 构建当前项目 | `--release`：使用release配置构建<br>`--target`：构建目标平台<br>`--jobs`：并行作业数<br>`--features`：启用指定特性 |
| `run` | 运行X语言程序 | `--release`：使用release配置<br>`--example`：运行指定示例<br>`--bin`：运行指定二进制目标 |
| `check` | 检查项目语法和类型（不生成代码） | `--all-targets`：检查所有目标 |
| `compile` | 编译X语言源代码到目标文件/可执行文件 | `--output`：指定输出文件<br>`--emit`：输出中间结果<br>`--no-link`：仅生成目标文件，不链接 |
| `test` | 运行项目测试 | `--release`：使用release配置<br>`--lib`：仅测试库<br>`--doc`：运行文档测试<br>`--no-run`：仅编译不运行 |
| `bench` | 运行项目基准测试 | `--no-run`：仅编译不运行 |
| `clean` | 清除构建产物 | `--doc`：仅清除文档<br>`--release`：仅清除release目录 |
| `doc` | 生成项目文档 | `--open`：在浏览器中打开<br>`--no-deps`：不生成依赖的文档<br>`--document-private-items`：包含私有项的文档 |
| `fetch` | 获取依赖（不构建） | - |
| `fix` | 自动修复代码中的警告 | `--allow-dirty`：允许在有未提交更改时修复<br>`--allow-staged`：允许在有暂存更改时修复 |

#### 依赖管理命令

| 命令 | 描述 | 主要选项 |
|------|------|----------|
| `add` | 添加依赖到x.toml | `--dev`：添加为开发依赖<br>`--build`：添加为构建依赖<br>`--optional`：标记为可选依赖<br>`--path`：使用本地路径<br>`--git`：从Git仓库添加 |
| `remove` | 从x.toml移除依赖 | `--dev`：从开发依赖移除<br>`--build`：从构建依赖移除 |
| `generate-lockfile` | 生成或更新x.lock | - |
| `locate-project` | 输出项目清单路径 | `--workspace`：查找工作区根 |
| `metadata` | 以JSON格式输出项目元数据 | `--no-deps`：不包含依赖解析 |
| `pkgid` | 输出完整的包标识符 | - |
| `tree` | 显示依赖树 | `--depth`：最大深度<br>`--invert`：反转依赖方向 |
| `update` | 更新x.lock中的依赖版本 | `--aggressive`：激进更新<br>`--dry-run`：试运行，不实际修改 |
| `vendor` | 将依赖复制到本地目录 | `--path`：输出目录<br>`--versioned-dirs`：使用带版本号的目录名 |

#### 项目管理命令

| 命令 | 描述 | 主要选项 |
|------|------|----------|
| `init` | 在当前目录初始化新项目 | `--lib`：创建库项目<br>`--vcs`：版本控制系统<br>`--edition`：X语言版本 |
| `new` | 创建新项目 | `--lib`：创建库项目<br>`--vcs`：版本控制系统<br>`--edition`：X语言版本 |
| `install` | 安装X语言可执行包（当前支持当前项目、本地路径、Git仓库与已配置的本地文件型注册表） | `--path`：从本地路径安装<br>`--git`：从Git仓库安装<br>`--version`：指定版本（暂不支持与 `--git` 一起使用）<br>`--list`：列出已安装的包 |
| `uninstall` | 卸载已安装的可执行包 | - |
| `search` | 在本地文件型注册表索引中搜索包 | `--limit`：最大结果数<br>`--registry`：指定注册表（远程注册表搜索尚未实现） |

#### 发布命令

| 命令 | 描述 | 主要选项 |
|------|------|----------|
| `login` | 保存包注册表令牌到本地配置 | `--registry`：指定注册表 |
| `logout` | 从本地配置中移除包注册表令牌 | `--registry`：指定注册表 |
| `owner` | 管理包的所有者（预留命令，尚未实现） | `--add`：添加所有者<br>`--remove`：移除所有者<br>`--list`：列出所有者 |
| `package` | 将项目打包为可分发的压缩包 | `--list`：列出将要打包的文件<br>`--no-verify`：跳过验证 |
| `publish` | 打包并验证项目，可发布到已配置的本地文件型注册表；远程发布仍在建设中 | `--dry-run`：试运行，不实际发布<br>`--no-verify`：跳过验证 |
| `yank` | 撤回已发布的版本（预留命令，尚未实现） | `--undo`：取消撤回<br>`--registry`：指定注册表 |

#### 工具命令

| 命令 | 描述 | 主要选项 |
|------|------|----------|
| `fmt` | 格式化X语言源代码 | `--check`：仅检查格式，不修改<br>`--all`：格式化所有文件 |
| `lint` | 代码检查（类似clippy） | `--fix`：自动修复<br>`--allow`：允许的lint<br>`--deny`：拒绝的lint<br>`--warn`：警告的lint |
| `repl` | 启动X语言REPL | `--target`：REPL目标平台 |
| `config` | 管理全局配置 | - |

## 构建系统

X语言的构建系统由 `x build` 命令驱动，它负责编译X语言源代码并生成可执行文件或库。构建系统的主要功能包括：

### 构建配置

- **开发模式**：默认构建模式，包含调试信息，优化级别较低，构建速度快
- **发布模式**：使用 `--release` 选项启用，优化级别高，生成的可执行文件更小、运行更快

### 构建目标

X语言支持多种构建目标，可以通过 `--target` 选项指定。构建目标决定了生成的代码将在哪个平台上运行。

### 依赖管理

构建系统会自动处理项目依赖，包括：
1. 解析 `x.toml` 文件中的依赖声明
2. 生成与展示本地依赖元数据
3. 构建项目源文件
4. 链接编译产物到最终的可执行文件或库

> 当前远程注册表下载与完整依赖获取流程仍在建设中；现阶段更可靠的方式是使用本地路径依赖、Git 依赖，以及已配置的本地文件型注册表 publish/search/install 流程。

### 并行构建

通过 `--jobs` 选项，可以指定并行构建的作业数，加快构建速度。

### 特性管理

X语言支持特性标志（feature flags），允许用户选择性地启用或禁用某些功能：
- `--features`：启用指定特性
- `--all-features`：启用所有特性
- `--no-default-features`：不使用默认特性

## 测试框架

X语言的测试框架由 `x test` 命令提供，它支持多种类型的测试：

### 单元测试

单元测试是最基本的测试类型，用于测试代码的各个组件是否正常工作。在X语言中，单元测试通常放在与被测试代码相同的文件中，使用 `test` 关键字标记。

### 集成测试

集成测试测试多个组件如何协同工作，通常放在项目的 `tests` 目录中。

### 文档测试

文档测试从代码文档中提取示例代码并执行它们，确保文档中的示例代码是正确的。使用 `--doc` 选项运行文档测试。

### 基准测试

基准测试用于测量代码的性能，使用 `x bench` 命令运行。

### 测试过滤

通过 `x test <filter>` 命令，可以只运行名称匹配指定模式的测试。

## 调试工具

X语言提供了多种调试工具，帮助开发者诊断和解决问题：

### REPL环境

`x repl` 命令启动一个交互式REPL（读取-求值-打印循环）环境，允许开发者直接执行X语言代码并查看结果。REPL支持两种目标平台：
- `interpreter`：使用X语言解释器（默认）
- `js`：编译为JavaScript并在JavaScript引擎中运行

### 代码格式化

`x fmt` 命令自动格式化X语言源代码，确保代码风格一致。它支持：
- 格式化单个文件：`x fmt <file>`
- 检查格式：`x fmt --check`
- 格式化所有文件：`x fmt --all`

### 代码检查

`x lint` 命令分析X语言代码，检测潜在的问题和不良实践。它可以：
- 自动修复某些问题：`x lint --fix`
- 配置lint规则：`--allow`、`--deny`、`--warn`

### 编译中间结果

`x compile` 命令的 `--emit` 选项可以输出编译过程中的中间结果，帮助开发者理解代码是如何被编译的：
- `tokens`：词法分析结果
- `ast`：抽象语法树
- `hir`：高级中间表示
- `pir`：低级中间表示
- `llvm-ir`：LLVM中间表示

## 工具链集成

X语言工具链已经能接入常见的现代开发工作流；当前最稳妥的范围仍是本地开发闭环、基础调试与已验证的 CLI 流程。它目前支持：

### 编辑器集成

- 与VS Code、IntelliJ等编辑器的集成
- 提供语法高亮、代码补全、错误检查等功能

### CI/CD集成

- 在CI/CD流水线中使用 `x check`、`x test` 等命令
- 支持自动化构建、测试和部署

### 包管理

- 通过 `x package` 生成可分发的 `.tar.gz` 压缩包
- 通过 `x install --path`、`x install --git` 或默认本地文件型注册表安装可执行项目
- 通过 `x search` 查询本地文件型注册表索引
- 通过 `x publish --registry <name>` 将包发布到已配置的本地文件型注册表
- 通过 `x login` / `x logout` 管理本地保存的注册表令牌

## 示例工作流

### 新建项目

```bash
# 创建新项目
x new my-project

# 进入项目目录
cd my-project

# 构建项目
x build

# 运行项目
x run

# 运行测试
x test

# 生成文档
x doc --open
```

### 添加依赖

```bash
# 添加依赖
x add serde

# 添加开发依赖
x add --dev test-framework

# 从Git仓库添加依赖
x add --git https://github.com/example/library
```

### 发布包

```bash
# 保存令牌到本地配置
x login <token>

# 打包当前项目
x package

# 试运行发布流程（执行打包与验证，但不会上传）
x publish --dry-run

# 发布到已配置的本地文件型注册表
x publish --registry local

# 从默认本地文件型注册表安装可执行项目
x install guessing_game
```

> 注意：当前远程注册表发布、远程注册表安装、所有者管理和版本撤回命令仍在建设中；上面的发布/安装示例仅适用于已配置的本地文件型注册表。

## 总结

X语言工具链当前已经提供项目初始化、构建、检查、运行、打包，以及基于本地路径、Git 或本地文件型注册表的可执行安装能力，同时提供本地文件型注册表的 publish/search/install 基础流程。命令行工具、构建系统、测试框架和调试工具已经形成了可用的本地开发闭环。

与此同时，远程注册表发布/安装、所有者管理、版本撤回，以及完整的依赖获取生态仍在持续建设中。现阶段更适合将 X 工具链视为一个正在走向生产价值的系统，而不是一个已经覆盖全部包生态流程的完成品。
